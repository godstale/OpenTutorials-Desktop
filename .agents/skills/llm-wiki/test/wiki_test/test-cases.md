# llm-wiki 스킬 테스트 케이스

이 문서는 `/wiki-query` 명령이 여러 메모를 교차 분석하여 복잡한 질문에 답할 수 있는지 검증하는 테스트 케이스를 정의합니다.

> **테스트 전제**: `raw2/` 폴더의 파일들을 `/wiki-ingest`로 위키에 먼저 추가한 후 `/wiki-query`로 아래 질문을 실행합니다.

---

## Test Case 1: 베트남 여행 잔여 예산 계산

### 사용 메모 파일 (raw2/)
- `베트남_여행_항공권.json` — 항공권 금액 (482,000원)
- `하노이_숙소_예약.json` — 숙소 5박 금액 (475,000원)
- `베트남_여행_예산_계획.json` — 총 예산 (1,600,000원) 및 여행 기간 (6박 7일)
- `하노이_공항_교통편.json` — 그랩 왕복 교통비 (34,000원)

### 질문
```
베트남 하노이 여행에서 항공권, 숙소, 공항 교통비(왕복)를 제외하면
관광·식사·쇼핑에 쓸 수 있는 잔여 예산은 얼마인가?
여행 기간 동안 하루에 평균 얼마를 쓸 수 있는지도 계산해줘.
```

### 기대 답변
- **고정 지출 합계**: 482,000원 (항공) + 475,000원 (숙소 5박) + 34,000원 (공항 교통 왕복) = 991,000원
- **잔여 예산**: 1,600,000원 − 991,000원 = **609,000원** (2인 합산)
- **1일 평균**: 609,000원 ÷ 7일 ≈ **87,000원/일** (2인 기준)
- 1인당 1일 평균 약 43,500원

### 왜 어려운가
- 총 예산은 `베트남_여행_예산_계획.json`에만 있음
- 항공권 금액은 `베트남_여행_항공권.json`에만 있음
- 숙박 금액은 `하노이_숙소_예약.json`에만 있음
- 교통비(왕복 = 편도×2)는 `하노이_공항_교통편.json`에서 계산해야 함
- 4개 파일의 숫자를 모두 읽고 산술 추론해야 정답 도출 가능

## Test Case 2: AI 개인 프로젝트 기술 격차 분석

### 사용 메모 파일 (raw2/)
- `AI_개인_프로젝트_목표.json` — 프로젝트 목표 및 필요 기술 스택
- `현재_보유_기술스택.json` — 현재 보유 기술과 숙련도
- `현재_학습중인_기술.json` — 현재 학습 중인 기술과 진도
- `AI_개발_학습_자료.json` — 보유 학습 자료 및 아직 못 찾은 자료

### 질문
```
내가 목표하는 AI RAG 지식관리 서비스를 만들기 위해
현재 보유 기술과 학습 현황을 종합했을 때,
지금 당장 시작하기 가장 어려운 부분(핵심 기술 격차)은 무엇이고
어떤 순서로 학습을 이어가는 것이 좋을까?
```

### 기대 답변
**핵심 기술 격차 (우선순위 순):**
1. **Vector DB (Chroma)**: 완전 미경험, 학습 자료도 아직 못 찾음 → RAG의 핵심이므로 가장 위험한 격차
2. **LangChain RAG**: 학습 중이지만 5장(RAG 챕터)이 미완료 상태
3. **FastAPI 중급**: 기초만 완료, 인증/비동기/배포 미학습
4. **Next.js**: 50% 완료, 백엔드 연동·인증 파트 미학습
5. **Docker Compose**: 기초 있음, 멀티 서비스 구성 경험 없음

**권장 학습 순서:**
1. LangChain 5장 RAG 챕터 마무리 (거의 완료 지점)
2. Chroma Vector DB 실전 예제 발굴 및 실습 (가장 큰 격차)
3. LangChain + Chroma + Ollama 통합 소형 프로토타입 구현
4. FastAPI 중급 (인증, 비동기) 병행 학습
5. Docker Compose로 전체 스택 통합

### 왜 어려운가
- 목표 기술 목록은 `AI_개인_프로젝트_목표.json`에만 있음
- 현재 보유 기술 수준은 `현재_보유_기술스택.json`에만 있음
- 학습 진도는 `현재_학습중인_기술.json`에만 있음
- 자료 공백(Chroma 예제 없음)은 `AI_개발_학습_자료.json`에만 있음
- 4개 파일을 대조해야 "무엇이 부족한지 + 얼마나 진행됐는지"를 동시에 판단 가능

## Test Case 3: 트레킹 장비 부족 분석

### 사용 메모 파일 (raw2/)
- `알틴아라샨_트레킹_계획.json` — 트레킹 조건 (고도 3,860m, 야간 -5°C, 3박 4일, 필요 장비 기준)
- `현재_보유_등산장비.json` — 현재 보유한 장비 목록 (미보유 장비 포함)
- `트레킹_준비물_체크리스트.json` — 필수 준비물 전체 목록

### 질문
```
7월 알틴아라샨 트레킹 (3박 4일 백패킹, 최고 고도 3,860m, 야간 최저 -5°C)을
위해 현재 보유한 장비로 해결되지 않는 필수 준비물은 무엇이고,
구매 우선순위를 알려줘.
```

### 기대 답변
**반드시 구매해야 할 장비 (우선순위 순):**

1. **-10°C 방한 슬리핑백** ★최우선
   - 보유: Silk liner 뿐 (보온 기능 없음)
   - 트레킹 조건: 야간 -5°C, 안전 마진 포함 -10°C 대응 필요
   - 미준비 시: 저체온증 위험

2. **55L 이상 백패킹 배낭**
   - 보유: Osprey 26L (당일 하이킹용), 40L 일반 백팩 (등산용 아님)
   - 3박 텐트+식량+장비 적재 시 26L로는 절대 불가

3. **방수 등산화 (발목 지지)**
   - 보유: Salomon 트레일화(방수 없음), 낡은 등산화(방수 마모)
   - 알라쿨 패스 구간 잔설·얼음 및 오후 소나기 대비 필수

4. **고어텍스 방수·방풍 재킷**
   - 보유: Columbia 레인재킷 (고어텍스 아님, 일반 방수코팅)
   - 오후 뇌우·강풍 상황에서 일반 방수코팅은 불충분

5. **헤드램프** (보유 없음)

6. **텐트** (보유 없음, 빌릴 계획이었다고 하나 확인 필요)

7. **응급 구급키트** (보유 없음) + **버너/코펠** + **워터필터**

### 왜 어려운가
- 트레킹 조건과 필요 장비 기준은 `알틴아라샨_트레킹_계획.json`에만 있음
- 현재 보유 장비 상태는 `현재_보유_등산장비.json`에만 있음
- 전체 필수 목록 기준은 `트레킹_준비물_체크리스트.json`에만 있음
- 3개 파일을 교차해야 "트레킹 조건 기준으로 보유 장비의 충족 여부"를 판단 가능

## Test Case 4: 이번 주 집중 작업 스케줄 최적화

### 사용 메모 파일 (raw2/)
- `이번주_할일_목록.json` — 이번 주 처리해야 할 작업 목록 (소요 시간, 작업 성격)
- `진행중_프로젝트_마감일.json` — 마감일 및 긴급도
- `집중력_시간대_패턴.json` — 시간대별 집중력 패턴 (창작 vs 코딩 특성 포함)
- `주간_고정_일정.json` — 이번 주 요일별 고정 일정

### 질문
```
이번 주 마감이 있는 작업들을 집중력이 가장 높은 시간대에 배치하고
고정 일정을 피해서, 요일별 최적 작업 스케줄을 만들어줘.
마감 순서와 작업 성격(창작/코딩)도 고려해줘.
```

### 기대 답변
**마감 우선순위:**
1. MCP 서버 테스트 → 수요일(4/30) 오후 6시 (★★ 코딩 작업)
2. 책 원고 3장 → 다음 주 월요일(5/4) 오전 10시 (★★★ 창작 작업)
3. 블로그 포스팅 → 금요일(5/2) 자정 (★ 글쓰기)
4. LangChain 공부 → 다음 주 수요일(5/6) 발표 (낮음)

**권장 주간 스케줄:**
- **월(4/28)**: 오전 9~10시 책 원고 집필 (고정 미팅 전 골든타임 활용), 오후 팀미팅·면담 소화, 저녁 9~11시 MCP 버그 수정 1차
- **화(4/29)**: 오전 9~11시 책 원고 집필 (최고 집중 + 창작), 오후 2~4시 MCP 테스트, 저녁 8~10시 독서모임 (고정)
- **수(4/30)**: 오전 9시 스크럼 후 10시~오후 2시 MCP 버그 수정 마무리 + 결과 공유 (6시 마감), 저녁 9~11시 블로그 초안
- **목(5/1)**: 공휴일 가족 외출 (10시~17시). 저녁 9~11시 책 원고 집중 집필 (마감 전날)
- **금(5/2)**: 오전 9~10시 블로그 포스팅 마무리, 오전 10~11시 주간 리뷰 미팅, 이후 LangChain 공부, 오후 14~15시 사이드 통화

### 왜 어려운가
- 작업 성격(창작/코딩)은 `이번주_할일_목록.json`에만 있음
- 마감일과 긴급도는 `진행중_프로젝트_마감일.json`에만 있음
- 어떤 시간대에 어떤 작업이 맞는지는 `집중력_시간대_패턴.json`에만 있음
- 피해야 할 고정 일정은 `주간_고정_일정.json`에만 있음
- 4개 파일을 모두 교차해야 최적 스케줄 도출 가능
