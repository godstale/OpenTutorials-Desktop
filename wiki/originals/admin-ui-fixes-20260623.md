# Admin Page UI Fixes & Error Handling (2026-06-23)

## 작업 개요
Admin 대시보드 및 관리 페이지에서 발생하는 각종 API 로딩 실패 오류를 수정하고, 사용자 요청에 맞춰 불필요한 UI 요소를 제거하거나 비활성화 처리하였습니다.

## 주요 수정 사항

### 1. API 로딩 실패 에러 및 500 오류 억제
- **문제**: `/api/admin/users`, `/api/admin/stats`, `/api/admin/macros` 호출 시 로컬 환경이나 DB 스키마가 없는 경우 500 서버 에러를 반환하여 프론트엔드에서 붉은색 토스트 경고("로딩 실패")가 표시됨.
- **해결**: 에러 발생 시 500 오류를 던지지 않고, 콘솔에 에러만 기록한 뒤 빈 배열(`[]`) 또는 기본값(`0`)을 반환하도록 수정.
- **결과**: 대시보드 화면에 가입자 및 에이전트 개수가 0으로 정상 표시되며, 사용자/매크로 관리 화면에서도 더 이상 에러 토스트가 뜨지 않음.

### 2. AI 에이전트 관리 페이지 UI 정리
- **문제**: 어드민에서 에이전트를 직접 제어할 필요가 없음.
- **해결**: `app/admin/agents/page.tsx` 테이블에서 '제어' 열과 관련된 조작 버튼(시작/중지)을 삭제하여 상태 모니터링만 가능하도록 변경.

### 3. 미사용 사이드바 메뉴 비활성화
- **문제**: '서비스 관리'와 '결제 관리'는 현재 사용되지 않음.
- **해결**: `components/admin/AdminSidebar.tsx`에서 두 항목을 필터링하여 비활성화(disabled) 상태로 하단에 분리하고, 그 위에 구분선(`SidebarSeparator`)을 추가함.

### 4. 서비스 설정 페이지 미구현 안내 추가
- **문제**: 설정 화면의 내용이 정해지지 않음.
- **해결**: 기존의 폼 UI를 모두 삭제하고, 옅은 노란색(amber) 테마의 미구현 경고 배너(`Alert`) 컴포넌트를 사용하여 사용자가 미구현 상태임을 인지하도록 안내.

## 결과 및 영향
- 불필요한 오류 알림이 제거되어 Admin 페이지 전반의 사용자 경험(UX) 개선
- 미사용 메뉴와 기능에 대한 접근을 시각적으로 차단하여 혼동 방지
- 백엔드 오류에 대한 프론트엔드의 graceful fallback 처리로 안정성 강화

### 5. CI 빌드 에러 해결 및 공통 컴포넌트 추가
- **문제**: 미구현 배너 작업에 사용된 `Alert` 컴포넌트가 프로젝트 내에 존재하지 않아 Vercel CI 빌드 과정에서 `Module not found` 에러 발생.
- **해결**: `references/registry-starter` 디자인 템플릿에서 `alert.tsx`를 비롯한 유용한 컴포넌트들(`accordion.tsx`, `popover.tsx`, `scroll-area.tsx`, `hover-card.tsx`, `form.tsx`)을 실제 컴포넌트 폴더로 복사하고, 연관된 패키지 종속성(`@radix-ui/*`)을 설치함.
- **결과**: 빌드 에러 해결 및 향후 개발에 활용할 수 있는 UI 기초 컴포넌트 환경 개선.
