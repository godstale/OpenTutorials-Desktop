# 강좌 학습 화면 AI 튜터 "히든 메시지" 파싱 및 자료 준비 상태 UI 연동

## 개요
AI 튜터가 사용자와 대화하면서 페니프레스 웹 앱 백그라운드로 처리해야 하는 제어/상태 메세지를 안전하게 전달하기 위한 **히든 메시지(Hidden Message)** 기술 표준을 수립하고, 이를 활용해 강좌 학습 화면에서 강좌 자료 파일의 실제 다운로드 완료 여부를 실시간으로 수신하여 튜터 헤더 UI 상태창에 반영했습니다.

## 배경 및 필요성
- **배경**: AI 튜터는 외부 자율 에이전트 서비스이므로 백그라운드에서 강좌 파일(resource.md)을 다운로드하고 인덱싱하는 작업을 개별적으로 진행합니다.
- **문제**: 사용자가 학습 시작 후 에이전트가 내부적으로 다운로드를 완료했는지 알 수 있는 방법이 없어 질문 시점 조율이 어려웠고, 백그라운드 제어 명령(JSON)이 사용자 대화방에 지저분하게 드러날 우려가 있었습니다.
- **해결**: HTML 주석 형식을 빌린 구조적 JSON 데이터 전송 프로토콜인 **히든 메시지**를 적용하여 백그라운드 상태 통신을 분리하고 화면에 깨끗하게 반영했습니다.

## 상세 구현 사항

### 1. 히든 메시지(Hidden Message) 프로토콜 정의
HTML 주석(`<!-- ... -->`) 안에 JSON 데이터를 포함시킵니다.
```html
<!-- HIDDEN_MESSAGE: {"action": "download_status", "downloaded": true} -->
```
- **이점**: 표준 마크다운 및 HTML 렌더러는 주석 처리를 기본적으로 시각화하지 않아 무해하며, 프론트엔드에서는 손쉽게 Regex를 활용하여 필요한 페이로드만 파싱하고 본문에서 숨길(strip) 수 있습니다.

### 2. 클라이언트 메시지 파싱 및 필터링
[app/(user)/learn/[slug]/client.tsx](file:///C:/Workspace/Projects/PennyPress-FE/app/(user)/learn/[slug]/client.tsx) 내에 다음 전처리 함수를 적용했습니다.
- **실시간 주석 가리개 (`cleanStreamingText`)**:
  스트리밍 중인 텍스트에 열린 주석 `<!--`가 존재하지만 아직 닫는 주석 `-->`을 받지 못한 경우, 텍스트 버퍼에서 해당 열린 주석부터 끝까지를 일시적으로 감추어 사용자 화면에 파편 코드가 노출되지 않도록 제어합니다.
- **정규식 메시지 정제 (`parseHiddenMessages`)**:
  스트리밍 데이터 수신 혹은 완료 후 정규식 `/<!--\s*HIDDEN_MESSAGE:\s*(\{[\s\S]*?\})\s*-->/g`을 매칭하여 JSON을 안전하게 꺼냅니다. `action === 'download_status'`일 경우 다운로드 boolean을 추출해 `courseDownloadStatus` 리액트 상태를 변경하고, 원본 말풍선 내용에서는 해당 주석 코드를 공백(`''`)으로 치환하여 제거합니다.

### 3. AI 튜터 시스템 프롬프트 업데이트
- **시스템 점검 질문(`checkPrompt`)**: 최초 페이지 진입 시 AI 에이전트에 보내는 시스템 진단 질문의 지시문을 보강하여, 점검 완료 시 답변 마지막에 `<!-- HIDDEN_MESSAGE: ... -->` 형태의 완료 상태 주석을 직접 반환하도록 안내했습니다.
- **시스템 역할 안내(`systemPrompt`)**: 기본 시스템 롤에 항상 자료 다운로드 여부를 판독하여 백그라운드 히든 메시지를 출력하라는 프롬프팅 규율을 추가했습니다.

### 4. 미려한 자료 준비 상태 UI 구현
우측 AI 튜터 사이드바 헤더에 에이전트의 연결 상태 밑으로 **자료 준비 상태 배지**를 실시간으로 노출했습니다.
- **자료 준비 완료**: 녹색 체크 링 테마 배지 (`bg-emerald-500/10 text-emerald-600 border-emerald-500/20`)
- **자료 준비 중 / 체크 중**: 호박색(Amber) 펄스 배지 및 로딩 스피너 (`Loader2` 컴포넌트 회전 애니메이션)
- **다운로드 오류**: 호박색(Amber) 경고 배지

## 파급 효과 및 의의
- **백그라운드 통신 표준화**: 차후 에이전트가 다른 백그라운드 트리거나 유저 정보 동기화 등을 원격 제어할 수 있는 채널로 확장 가능합니다.
- **UX 극대화**: 로딩 및 상태 전이를 HSL 테마에 어우러지는 모던하고 유려한 애니메이션으로 표현하여 사용자가 에이전트의 구동 상태를 쉽게 확인하도록 도왔습니다.
- **데이터 흐름 격리**: 사용자에게 닿는 가독성 높은 답변 텍스트와 프론트엔드가 삼키는 백그라운드 데이터의 분리 샌드박싱에 성공했습니다.
