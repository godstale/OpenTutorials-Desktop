# PennyPress AI Tutor 강좌 학습 연동 및 숨김 콘텍스트 조합 프롬프트 구현

**작업일자:** 2026-06-28

## 1. 개요
강좌 학습 화면에서 특정 외부 에이전트를 "AI 튜터(강좌 학습용 에이전트)"로 지정하여 학습 지도를 받을 수 있는 기능을 연동하고, 질문 시 프롬프트에 강좌 전체 내용 및 현재 학습 카드 내용을 숨김 처리된 콘텍스트로 포함하여 답변 품질을 극대화하도록 개선했습니다.

## 2. 주요 구현 내용

### 2.1. 데이터베이스 스키마 확장 및 타입 정의
- `user_external_agents` 테이블에 `is_ai_tutor` BOOLEAN 컬럼(기본값 `FALSE`) 추가 (`supabase/migrations/20260701_add_is_ai_tutor_to_external_agents.sql`, `docs/supabase_setup_query.txt`).
- TypeScript 인터페이스 `UserExternalAgent`에 `is_ai_tutor?: boolean;` 프로퍼티 추가 (`lib/types/index.ts`).

### 2.2. 배타적 단일 선택 비즈니스 로직
- `lib/api/external-agents.ts`의 `createExternalAgent` 및 `updateExternalAgent` 함수를 보완하여, 하나의 에이전트가 AI 튜터로 활성화(`is_ai_tutor` = `true`)될 경우, 해당 유저의 기존 다른 모든 에이전트들의 `is_ai_tutor` 값을 일괄 `false`로 리셋함으로써 단일 에이전트 지정 보장을 백엔드 차원에서 구현했습니다.

### 2.3. 외부 에이전트 UI 업데이트
- `components/features/AddAgentModal.tsx` 및 `components/features/AgentSettingsTab.tsx`에 **AI 튜터 설정용 스위치 토글**을 추가하였습니다.
- 조회 모드에서 `"AI 튜터 지정 여부: 지정됨 / 미지정"` 상태 정보를 제공합니다.
- `app/(user)/my-agents/page.tsx` 목록 화면의 에이전트 카드 상단에 가시성이 높은 배지(`AI 튜터`)를 표시합니다.

### 2.4. 학습 플레이어 내 AI 튜터 연동 및 숨김 프롬프트
- `app/(user)/learn/[slug]/client.tsx`에서 `is_ai_tutor`가 `true`로 설정된 에이전트만 AI 튜터로 매핑합니다. 튜터로 지정된 에이전트가 없으면 상태를 `none`으로 설정해 사이드바에 `"미연동 - 설정 필요"` 가이드 메시지를 올바르게 노출합니다.
- 질문 전송 시 백엔드 API 호출에는 **강좌 전체 내용 및 각 카드 내용(System Message)**과 **현재 학습 중인 카드의 내용(Context Prefix)**을 조합하여 프롬프트로 전송합니다.
- 사용자의 채팅창 화면(`messages` 상태) 및 데이터베이스 대화 이력(`user_external_agent_messages`)에는 거대한 콘텍스트가 생략된 **순수한 원본 질문과 AI 답변**만 깔끔하게 저장 및 노출되도록 필터링 처리하여 사용자 경험과 데이터베이스 공간을 동시에 효율화했습니다.

## 3. 프롬프트 및 에이전트 연동 추가 개선 (2026-06-28 2차 개선)

### 3.1. 강좌 리소스 다운로드 API (`/api/courses/[slug]/resource`) 도입
- 기존에는 매번 첫 질문 전송 시 강좌 전체 텍스트(Metadata + 전체 카드 내용)를 시스템 프롬프트에 직접 포함하여 전송했으나, 이는 큰 API 오버헤드와 중복 토큰 낭비를 유발했습니다.
- 이를 해결하기 위해 지정된 강좌 슬러그의 요약, 목차, 그리고 전체 카드 본문 텍스트를 컴파일하여 단일 Markdown 문서로 내려주는 리소스 전송 API 엔드포인트를 구현했습니다.
- 외부 에이전트(예: Hermes Agent)가 이 URL을 통해 자원을 직접 가져갈 수 있도록 `lib/supabase/proxy.ts` 미들웨어 단에서 인증 예외 처리를 추가하여 비인증 외부 요청을 가능하게 하였습니다.

### 3.2. AI 튜터 핑 동기화 및 오프라인 대기 고착 장애 해결
- 페이지 진입 시 데이터베이스의 이전 상태만을 읽어와 연결이 활성화되어 있음에도 화면상에 계속 "오프라인 - 연결 대기중" 상태로 고착되는 현상을 수정했습니다.
- 강좌 학습 화면 마운트 시 `/api/external-agents/test` API로 동적 핑(ping) 테스트를 수행하고, 감지된 실제 에이전트 상태에 맞춰 데이터베이스 및 로컬 상태를 실시간 갱신(updateExternalAgent)하도록 동기화 로직을 개선했습니다.

### 3.3. 프롬프트 캐싱 및 다운로드 유도 지침 적용
- 첫 번째 턴에서 거대한 텍스트 본문을 전송하는 무거운 방식을 지양하고, 대신 전체 강좌 리소스 URL과 함께 "이미 세션 혹은 에이전트 워크스페이스 내에 다운로드/캐싱되었다면 재다운로드하지 말라"는 지침만 프롬프트를 항상 전송하도록 경량화하였습니다.
- 이로 인해 첫 턴부터 마지막 턴까지 대화 페이로드 크기가 항상 1KB 미만으로 최소화되었으며, 불필요한 네트워크 트래픽 및 토큰 소비가 획기적으로 개선되었습니다.
