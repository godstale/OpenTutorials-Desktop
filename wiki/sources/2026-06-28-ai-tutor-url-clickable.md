# AI 튜터 답변 내 URL 감지 및 클릭 시 새 창 링크 연동

## 개요
AI 튜터의 답변 내용 중에서 웹 사이트 주소(URL) 또는 마크다운 형식의 링크를 실시간으로 판독하여, 사용자가 클릭했을 때 브라우저의 새 창(새 탭)에서 안전하게 열릴 수 있도록 하는 **URL 링커(Linker) 컴포넌트 기능**을 구현했습니다.

## 배경 및 필요성
- **배경**: AI 튜터가 강좌 내용에 대한 답변을 작성하면서 외부 참고 문서나 관련 링크(예: Supabase 가이드, GitHub URL 등)를 제공하는 경우가 잦습니다.
- **문제**: 이전 파서 구조에서는 말풍선 내 텍스트에 URL 주소가 평문 문자열로 그대로 출력되어 복사-붙여넣기해야 하거나, 마크다운 링크 형식(`[label](url)`)의 코드가 노출되어 클릭할 수 없고 가독성이 손상되는 문제가 있었습니다.
- **해결**: 답변 문자열 렌더링 영역인 `ChatMessageContent` 내부에 정교한 URL 링크 추출기인 `renderTextWithLinks`를 연동하여, 사용자 경험과 정보 접근 편의성을 비약적으로 높였습니다.

## 상세 구현 사항

### 1. 하이브리드 링크 추출기 (`renderTextWithLinks`)
웹 표준 URL 패턴과 마크다운 링크 문법을 모두 수용하는 하이브리드 추출 함수를 설계했습니다.
- **마크다운 링크 감지 (`[label](url)`)**:
  - 정규식 `(\[[^\]]+\]\((?:https?:\/\/[^\s)]+)\))`을 사용해 텍스트에서 마크다운 형태의 링크를 분리합니다.
  - 해당 마크다운 코드가 포착되면 라벨 텍스트와 실제 URL을 분리하여 리액트 `<a>` 엘리먼트로 동적 변환합니다.
- **일반 평문 URL 감지 (`https://...`)**:
  - 텍스트에서 `https?://`로 시작하는 공백 및 문장 부호 이전의 주소를 감지합니다.
  - 정규식 `(https?:\/\/[^\s<>\(\)\[\]"'`]+)`를 적용하여 백틱, 작은/큰 따옴표, 괄호 등으로 감싸인 URL도 정확하게 주소 영역만 추출합니다.
- **끝 문장 부호 예외 처리**:
  - 문장 종결에 따라 URL 끝에 붙는 마침표(`.`), 쉼표(`,`), 느낌표(`!`), 물음표(`?`), 콜론(`:`) 등을 정규식 `/[.,!?:]+$/`으로 감지해 링크 타겟 주소에서 제거하고, 일반 텍스트 형태로 주소 바로 옆에 다시 붙여줌으로써 링크가 잘못 빌드되는 오류를 차단했습니다.

### 2. 마크다운 스타일 데코레이터 통합
- **볼드체 및 일반 텍스트 결합**:
  - 마크다운 파서가 작동하는 과정에서 볼드 표시(`**text**`) 내부나 외부 텍스트 영역 어디서든 링크가 렌더링되도록 결합했습니다.
- **스타일링 일관성**:
  - Shadcn/Tailwind CSS 디자인 가이드에 맞추어 `text-primary hover:underline font-medium break-all` 클래스를 부여하여 가독성과 클릭 피드백을 강화했습니다.
  - `target="_blank" rel="noopener noreferrer"` 속성을 고정 부여하여 외부 사이트로 연결 시 기존 서비스 페이지가 덮어씌워지지 않고 독립된 브라우징 컨텍스트에서 열리게 보안성을 충족했습니다.

## 기대 효과
- **사용자 정보 접근 속도 향상**: 외부 참조나 개발자 문서 링크를 원클릭으로 신속하게 탐색할 수 있어 학습 효율이 향상됩니다.
- **완성도 높은 마크다운 렌더러**: 외부 에이전트가 흔히 생성하는 마크다운 링크 서식을 프론트엔드가 말끔하게 변환 수용함으로써 UI 안정성이 향상됩니다.
- **보안 격리**: `rel="noopener noreferrer"` 조치를 완비하여 탭 가로채기(Tabnabbing)와 같은 피싱 공격 경로를 차단했습니다.
