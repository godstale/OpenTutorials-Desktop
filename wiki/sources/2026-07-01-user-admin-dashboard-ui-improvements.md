# 사용자 및 관리자 대시보드 UI 개선, 강좌 검색 및 나의 강좌 카드 이동성 및 여백 개선

- **날짜**: 2026-07-01
- **작성자**: Antigravity
- **상태**: 완료 (Completed)

---

## 📌 개요

PennyPress 플랫폼의 사용자 포털 및 관리자 대시보드 편의성과 일관성을 개선하기 위해 아래 작업들을 수행했습니다.
1. **사용자 포털 > 대시보드 > 학습 중인 강좌**: 강좌/패키지 카드 중앙부를 클릭하면 상세 페이지로 연결되는 링크 추가, 하단 푸터 영역의 언밸런스한 하단 여백 해결 (`py-0`, `pb-3` 적용).
2. **사용자 포털 > 강좌 검색 > 종합 강좌 패키지**: 강좌 카드의 하단 푸터 영역의 언밸런스한 하단 여백 해결 (`py-0` 적용).
3. **종합 강좌 패키지, 개별 강좌 타이틀 옆 배지 삭제**: "SaaS 추천" 및 "개별 신청" 배지 삭제로 UI 단순화.
4. **사용자 포털 > 나의 강좌**: 강좌 카드 클릭 시 학습 화면 대신 강좌 상세 페이지(`/my-courses/[slug]`)로 이동하도록 수정, 우측 상단 `[새 강좌 찾기]` 버튼의 색상을 초록색(`bg-emerald-600`) 테마로 수정하여 일관성 확보.
5. **관리자 포털 > 대시보드**: 미사용 카드인 "등록 외부 에이전트" 및 "활성 매크로" 삭제, 대신 "강좌 패키지" 및 "개별 강좌" 등록 수치 요약을 표시하는 카드로 교체.

---

## 🛠️ 상세 구현 내용

### 1. 사용자 대시보드 강좌 카드 클릭성 및 여백 개선
- **수정 파일**: `app/(user)/dashboard/page.tsx`
- **구현**:
  - 카드 상단 썸네일, 제목(CardHeader), 본문(CardContent) 영역을 감싸는 `<Link>` 컴포넌트를 추가하여, 클릭 시 개별 강좌 상세 페이지(`/my-courses/[slug]`) 또는 패키지 상세 페이지(`/packages/[slug]`)로 즉시 이동할 수 있게 했습니다.
  - Card 컴포넌트에 default로 포함된 `py-6` 패딩을 무력화하기 위해 `py-0 pb-0`을 추가하여 썸네일과 하단 푸터 영역의 정렬 및 여백을 밀착시켰습니다.
  - CardFooter에 `pb-3` 패딩을 적용하여 상하 대칭 여백을 완성함으로써 언밸런스해 보이던 문제를 해결하였습니다.

### 2. 강좌 검색 화면 여백 및 배지 단순화
- **수정 파일**: `app/(user)/courses/page.tsx`
- **구현**:
  - 종합 강좌 패키지 카드에도 동일하게 `py-0`을 추가하여 `CardFooter` 하단에 불필요하게 남아돌던 공백을 제거하고 여백 정렬을 통일했습니다.
  - "종합 강좌 패키지" 타이틀 옆에 노출되던 `<Badge>SaaS 추천</Badge>`와 "개별 강좌" 타이틀 옆의 `<Badge>개별 신청</Badge>`를 완전히 제거하여 화면을 한결 정돈했습니다.

### 3. 나의 강좌 페이지 상세 이동 연동 및 버튼 개선
- **수정 파일**: `app/(user)/my-courses/page.tsx`
- **구현**:
  - 기존에는 개별 강좌 카드를 클릭하면 바로 학습 화면(`/learn/[slug]`)으로 진입했으나, "이어서 학습하기" 버튼과의 의미 중복을 해결하고 강좌 상세/목차를 먼저 보도록 카드의 주 `onClick` 핸들러 주소를 `/my-courses/${course.slug}`로 변경했습니다.
  - 화면 우측 상단의 `[새 강좌 찾기]` 버튼에 `bg-emerald-600 hover:bg-emerald-700 text-white border-none shadow-sm` 스타일을 부여하여 초록색 일관성을 적용하였습니다.

### 4. Admin 대시보드 요약 지표 개편
- **수정/신규 파일**:
  - `app/admin/dashboard/page.tsx` (지표 카드 UI 및 `AdminStats` 타입 변경)
  - `app/api/admin/stats/route.ts` (API 쿼리 및 응답 데이터 구조 변경)
- **구현**:
  - 기존의 에이전트 수("등록 외부 에이전트"), 크론 수("활성 매크로") 통계는 불필요하므로 제거했습니다.
  - 대신 데이터베이스의 `course_packages` 및 `courses` 테이블의 전체 레코드 개수를 카운트하도록 API Route를 재작성하였습니다.
  - 환경변수 및 DB 로컬 설정이 누락되어도 로딩 크래시나 에러 Toast가 발생하지 않도록 graceful try-catch 및 dummy fallback 데이터를 보강하였습니다.
