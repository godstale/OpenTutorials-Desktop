# 강좌 및 에이전트 관리 UI 개선 및 에이전트 대화 전체 보존(토큰/시간 로그 기록)

**작업일자:** 2026-07-05

## 1. 개요
강좌 관리 및 에이전트 관리 화면의 사용자 경험(UI/UX)을 직관적으로 개선하고 불필요한 장식용 뱃지나 버튼들을 정리하였습니다. 또한, 사용자와 에이전트가 나눈 대화 기록 전체를 영구 보존할 수 있도록 에이전트별 대화 로깅 시스템을 도입하고 소요 시간 및 입력/출력 토큰 크기를 정밀 기록하도록 고도화하였습니다.

## 2. 주요 구현 내용

### 2.1. 강좌 관리 UI 개선
- 강좌 패키지 리스트 화면(`app/(user)/courses/manage/page.tsx`)의 각 강좌에서 복잡도를 낮추기 위해 사용하지 않는 **[매니페스트 수정]** 버튼을 제거하였습니다.
- 버튼 삭제에 따라 2열 Grid 구조였던 액션 버튼 레이아웃을 유연한 Flex (`flex flex-wrap items-center`) 레이아웃으로 최적화하여 썸네일 우측 영역에 깔끔히 배치되도록 하였습니다.

### 2.2. 에이전트 관리 및 카드 UI 개선
- 에이전트 카드(`app/(user)/my-agents/page.tsx`)의 상단에서 의미가 불분명했던 **"지침 설정 완료/대기"** 뱃지를 삭제하였습니다.
- 카드 하단의 **[웹 UI]** 버튼을 제거하고, 에이전트 설정을 즉각 제거할 수 있는 **[삭제]** 버튼을 추가하였습니다.
- 에이전트 카드의 body(헤더 및 컨텐츠 영역)를 클릭하면 해당 에이전트의 상세 화면(`/my-agents/[id]`)으로 바로 라우팅되도록 개선하였으며, 카드 내부 버튼 및 드롭다운 클릭 시 상세 라우팅이 중복 트리거되지 않도록 `e.stopPropagation()` 처리를 완벽하게 적용하였습니다.
- 에이전트 페이지 최초 진입 시, 모든 에이전트 상태를 자동으로 핑(ping) 테스트하여 오프라인/온라인 연결 배지가 정확한 실시간 상태를 표시하도록 동기화하였습니다.

### 2.3. 에이전트 상세 화면 개선 및 탭 단일화
- 에이전트 포털 상세 화면(`app/(user)/my-agents/[id]/page.tsx`)의 탭 구성 중 불필요한 "칸반 / 웹 UI" 및 "원격 설정"을 제거하고 **[대화 (Chat)]**와 **[설정]** 두 개만 깔끔하게 유지하도록 간소화하였습니다.
- 대화 탭(`components/features/AgentChatTab.tsx`)의 헤더 영역에서 중복되던 **Model 정보** 표시를 삭제하였습니다.
- footer 입력창 위에 상주하던 복잡한 **[매니페스트/매크로 관련 버튼 및 검색 패널]**을 완전 제거하고, 텍스트 입력창 placeholder에서 매크로 관련 가이드를 삭제하여 순수 채팅 환경을 구축하였습니다.

### 2.4. 에이전트 이탈 시 5분 연결 종료 타이머 제어 (연결 끊김 방지)
- 에이전트 상세 화면에 사용자가 머물러 있는 동안에는 연결이 끊어지지 않고 상태를 영구적으로 유지하도록 기존의 3분 미입력(idle) 타임아웃 오프라인 강제 전환 기능을 삭제하였습니다.
- 에이전트 상세 화면(`app/(user)/my-agents/[id]/page.tsx`) 또는 학습 화면(`app/(user)/learn/[slug]/client.tsx`)을 이탈(unmount)할 때, 즉시 오프라인으로 갱신하는 대신 **5분(300,000ms) 타이머**를 가동하고, 5분이 경과했을 때만 명시적으로 연결을 오프라인(`status: 'offline'`)으로 전환하도록 개선하였습니다.
- 사용자가 페이지 전환 등으로 이탈했다가 5분 이내에 다시 상세 화면이나 학습 화면으로 복귀하는 경우, 기존에 가동 중인 이탈 타이머를 해제(`clearTimeout`)하여 연결이 유실되지 않고 매끄럽게 유지되도록 SPA 대응 전역 타이머 관리 맵(`agentLeaveTimers`)을 `lib/utils.ts`에 추가 구현하였습니다.

### 2.5. 에이전트 대화 전체 보존 및 통계/분석 로그 파일 기록
- 기존 100개 제한이 걸려있던 Supabase DB 대화 이력과 별개로, 에이전트와 나눈 모든 대화를 로컬에 누적 보존하기 위해 에이전트별 대화 로깅 시스템(`app/api/external-agents/[id]/chat/route.ts`)을 구현하였습니다.
- 사용자와 에이전트의 대화가 한 턴 완료(스트림 응답 종료)될 때마다, 다음 통계 지표를 실시간으로 측정하여 `public/agent-chats/<agent_id>.json` 파일에 JSON 배열로 기록 및 추가(append)합니다:
  - **대화 시각 (Timestamp)**
  - **대화 소요 시간 (duration_ms)**: 대화 API 호출 시작점부터 스트림 최종 응답 완료 시점까지의 차이
  - **입력/출력 토큰 사이즈 (input_token_size, output_token_size)**: 한글 글자수와 영문 단어수를 감안한 정교한 한글 가중치 근사 토큰 카운터 엔진 탑재
  - **사용자 원본 메시지 및 에이전트 최종 답변 본문**
- 생성된 대화 로그 파일은 Next.js 로컬 구동 시 누적 적재되어 추후 에이전트 답변 품질 분석 및 통계 대시보드 시각화의 기초 데이터로 영구히 활용됩니다.
