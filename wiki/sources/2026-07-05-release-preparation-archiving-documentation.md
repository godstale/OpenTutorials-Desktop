# Open Tutorials 배포 준비 및 아카이빙 작업 상세

**작업일자:** 2026-07-05

## 1. 개요
Open Tutorials의 오픈소스 릴리즈 및 배포 작업을 원활하게 수행하기 위해 프로젝트 구조를 클린업하였습니다. 기존 클라우드 기반 PennyPress의 잔재 폴더(파이썬 에이전트 워커, Supabase 마이그레이션, 위키 그래프 리소스 등)를 아카이빙하고, 새로 피벗된 로컬 온디바이스 학습 플랫폼에 맞춰 설치, 구동, 강좌 관리, 강좌 빌드 매뉴얼을 새롭게 작성 및 배치하였습니다.

## 2. 세부 작업 내용

### 2.1. 불필요 리소스 아카이빙 및 디렉토리 구조 정리
- **대상 폴더**:
  - `agent-worker/` (기존 로컬 개발/테스트용 파이썬 Mock 서버)
  - `supabase/` (기존 Supabase PostgreSQL DB 마이그레이션 스크립트)
  - `graph/` (위키 그래프 렌더링 HTML/JSON 정적 산출물)
- **조치**: 해당 디렉토리들을 `docs/archive/` 하위 경로로 일괄 이동하여 소스 빌드 및 일반적인 배포 대상에서 즉각 격리시켰습니다.

### 2.2. `.gitignore` 규칙 업데이트
- `docs/` 디렉토리 하위의 리소스 중, 배포에 필수로 포함되어야 하는 다음 두 개 디렉토리만을 예외 허용하고 나머지는 모두 무시하도록 룰을 최적화했습니다.
  - `!/docs/architecture/` (앱 구조 설명 문서)
  - `!/docs/manual/` (앱 사용 및 번들 가이드 문서)
- `.gitignore` 설정 내용:
  ```gitignore
  # Docs Configuration
  /docs/*
  !/docs/architecture/
  !/docs/manual/
  ```

### 2.3. 아키텍처 및 사용자/제작자 가이드 문서 작성
- **`docs/architecture/README.md` (신규)**:
  - Frontend, Local DB 에뮬레이션(`db.json` 및 `lib/supabase/mock-client.ts`), 정적 파일 스토리지 모킹, AI 튜터 통합 아키텍처 및 권한/세션 바이패스 방식을 기술하여 개발자들이 구조를 파악할 수 있도록 도왔습니다.
- **`docs/manual/README.md` (신규)**:
  - 사용자를 위한 앱 로컬 설치/구동, 강좌 등록 및 AI 튜터 활용법을 상세히 기술하였습니다.
  - 제작자를 위한 통합 번들 ZIP 구조와 필수 정합성 규칙을 샘플 JSON 파일 구조와 함께 포함하였습니다.
  - 강좌 마크다운 카드와 메타데이터 패키징을 자동화하기 위한 AI 에이전트 도구인 `OpenTutorials-Bundler` 프로젝트(GitHub 주소 포함) 사용 안내를 연동하였습니다.

### 2.4. 루트 `README.md` 최신화
- PennyPress 기반의 복잡한 SaaS 인프라 설정(Vercel, Supabase Auth, Cloud API 등) 내용을 전면 삭제하고, Open Tutorials 온디바이스 플랫폼 설명, 신규 작성한 `docs/` 매뉴얼 링크, 로컬 실행을 위한 초간단 단계를 기재하여 직관성을 극대화하였습니다.

## 3. 결과 및 기대효과
- 릴리즈 배포용 패키지 빌드 시 불필요한 의존성 파일 및 개발용 임시 데이터가 유입되는 것을 미연에 방지합니다.
- 사용자와 콘텐츠 제작자가 로컬 플랫폼을 구동하고 스스로 강좌 ZIP 패키지를 빌드/업로드하여 학습하는 일련의 라이프사이클을 완벽하게 가이드합니다.
